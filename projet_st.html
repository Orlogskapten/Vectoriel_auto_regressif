1). Explain what a VAR model and an impulse response function is.¶

Le modle VAR permet danalyser des sries temporelles de manire multivaries, en tudiant les dpendances linaires au sein de toutes les sries que l'on considre. Il se distingue d'un modle AR par cet aspect multivari.

Pour expliquer une variable $X_t$, avec un modle VAR d'ordre 1, nous allons utiliser la donne de la priode prcdente telle que :

$$VAR(1): X_t= \phi_0 + \phi_1X_{t-1} + \epsilon_t$$

avec : $$X_t= \begin{pmatrix} chine\_growth_t\\ USA\_growth_t\\ Zone\_growth_t \end{pmatrix}$$

A l'ordre p on a: $$VAR(p): X_t= \phi_0 + \phi_1X_{t-1} + \phi_2X_{t-2} + ... + \phi_pX_{t-p} + \epsilon_t$$ C'est--dire qu'un VAR d'ordre p permet de considrer un lien entre nos donnes en t et les donnes observes jusqu'en t-p.

Ce qu'il y a de magique avec un modle VAR, c'est qu'on peut le transformer et faire disparatre les sries $X_{t-p}$ pour ne faire apparatre que les chocs. L'ide derrire cette transformation est de pouvoir expliquer les valeurs de $X_t$ en fonction des chocs passs. En conomie, c'est un concept utile par exemple pour voir comment un choc de demande se propage travers les sries de taux de croissance, taux d'intrt et de taux de chmage ; car en conomie les variables macroconomiques ne sont que rarement indpendantes les unes des autres, et donc observer et comprendre l'impact d'un choc provenant d'une srie sur les autres est essentiel.

Dans notre projet, nous avons les taux de croissance de la Chine, des Etats-Unis et de la Zone Euro de 1996 fin 2019. Avec la mondialisation, et les connexions de plus en plus grandes entres les conomies, il est intressant de voir comment un choc de croissance dans un pays, pourrait en atteindre un autre et surtout, pendant combien de temps. Ce n'est pas sans rappeler la situation actuelle et la crise mondiale du COVID qui touche toutes nos conomies. Mais on aura l'occasion d'en parler plus longuement dans la dernire question.

Pour coder un VAR d'ordre p, il nous a fallu repenser la construction du dataset, et de son interaction avec les coefficients optimiser.

Imaginons que l'on doive construire un VAR d'ordre 2. Le dataset que nous devrions utiliser pour le construire ressemblerait ceci :

mvnorm<-function(X,mu,sigma)
{
    # Permet de calculer la fonction densit
        # X reprsente un vecteur 1xn
        # Mu reprsente un vecteur 1xn
        # Sigma une matrice nxn
    A=(2*pi)^(ncol(sigma)/2)
    B=det(sigma)^(1/2)
    C=-1/2*t(X-mu)%*%solve(sigma)%*%(X-mu)
    D=exp(C)
    return(1/(A*B)*D)
}

VAR_loglik_p<-function(para, vectored)
{
    # Permet de calculer la log vraisemblance d'un modle VAR(p)
        # para dsigne un vecteur de (n + n*n*p) contenant les paramtres du modle
        # vectored correspond  un vecteur contenant le datatset, l'ordre du VAR, et les dimensions du datatset 
    
    # Rcupre l'information du vecteur
    stocked= tail(vectored, 3)
    p= stocked[3]
    n= stocked[1]
    m= stocked[2]
    X= matrix(vectored[1: (length(vectored) - 3)], m, n )
    
    # Extraction des intercepts
    phi_0= para[1:n]
    # E dsigne la valeur X calcule  l'aide du modle 
    # On construit en amont E, ce qui nous permet d'ajouter les intercepts, et de bien dfinir
    # sa dimension
    E= matrix(phi_0, m-p, n, byrow= T)
    
    # Si l'ordre du VAR = 3, alors il y aura 3 matrices de dimension nxn
    # On rcupre par itration les coefficients qui sont associs  chaque matrice (en premier 
    # la matrice associe aux donnes avec le premier retard, puis le deuxime etc.)
    for (i in 1:p){
        # Rcupre les coefficients de la matrice du retard i
        phi_i= matrix(para[((n*n*i + n) -(n*n) + 1):(n*n*i + n)], n ,n)
        # Pour la matrice phi_1, les coefficients phi1_11, phi1_12, phi_13 dans le cas d'une var
        # avec 3 sries, ne seront pas en ligne comme dans une reprsentation matricielle
        # mais seront stocks dans la premire colonne !!

#         E= E[-1,] + X[-c((m-i+1):m),]%*%phi_i # enlve le bas
#         E= E[-dim(E)[1],] + X[-c((m-i+1):m),]%*%phi_i # enlve le bas
        
        # On fait le calcul phi_p . , mais comme les sries de X sont stockes en ligne
        # et que les coefficients sont sotcks en colonne, on doit faire X . phi_p
        # On enlve une partie de la matrice (le bas) qui dpend de l'ordre sur lequel on itre
        # cf le markdown ?
        phi_compute= X[-c((m-i+1):m),]%*%phi_i # enlve le bas de la matrice X pour associer les bons retards
        
        if (i == p){
            E= E + phi_compute
        }
        else {
            E= E + phi_compute[-c(1:(p-i)),] # enlve le haut pour que les retards fit bien avec E et X
        }
    }
    
    # Pour concorder avec le retard max (= ordre p), on doit se sparer des p premires lignes de X
    residus= X[-c(1:p), ] - E
    sigma= var(residus)
    
    log_lik= 0
    
    # Calcul de la log vraisemblance
    # On commence la boucle  p+1 et non  1 pour simplifier le raisonnement (permet de
    # slectionner les donnes X  partir de i)
    # Mais on aurait pu commencer  1 et on aurait modifier l'indice dans X et E
    for (i in (1+p):m){
        temp= mvnorm(X[i, ], E[(i-p),], sigma) # E est pris  partir de p car j'ai enlev p lignes
                                                # dans le processus prcdent
        temp= log(temp)
        log_lik= log_lik - temp
    }
    
    return(log_lik)
}

2). Using information criterions, estimate the lag to be used with the three data series for your VAR model. What do you think of this lag?¶

Comme il est possible de gnrer p VAR, il nous faut dterminer l'ordre qui est le plus intressant pour modliser notre srie de donnes. Dans ce but, nous allons utiliser les fonctions prcdentes pour construire tous les modles VAR de l'ordre 1 10. Pour comparer tous ces modles, nous allons utiliser des critres d'informations (AIC, BIC et HQ), qui permettent de prendre en compte la fois la performance d'un modle (la valeur de la log vraisemblance) mais aussi sa complexit (le nombre de paramtres). En effet, il est trs simple d'avoir une bonne log vraisemblance en augmentant le nombre de paramtres. Mais le modle devient trop spcifique notre jeu de donnes. Dans le cas d'un VAR, si on prend un ordre trs lev, le nombre de paramtres sera alors plus grand qu'avec un VAR d'ordre 1. En effet, si n reprsente le nombre de sries, le nombre de paramtres d'un VAR d'ordre p sera alors de $n*n*p + n$.

En cherchant minimiser les critres d'informations, on trouve le modle qui a un bon quilibre entre performance et complexit.

Dans notre cas, on a reprsent la valeur de ces critres dans le tableau ci-dessous.

formule_generale= function(p, log_lik, n, m, g){
    # Permet de calculer tous les critres d'informations en fonction de g
        # P dsigne l'ordre du VAR
        # log_lik dsigne la log vraisemblance du modle calcule
        # n le nombre de sries (permet de calculer le nombre de paramtres du modle)
        # m le nombre d'observations
        # g correspond  la fonction d'information slectionne
    base= -(2*log(log_lik))
    k= n + n*n*p # nombre de param
    return (base + (k*g))
}

bic_g= function(m){
    # Permet de calculer la fonction g pour le critre BIC
    return (log(m))
}

hq_g= function(m){
    # Permet de calculer la fonction g pour le critre HQ
    return (log(bic_g(m)))
}

Aussi, on pensait que chacune des sries taient autocorrles sur plusieurs priodes (6 mois voire 1 an). Par consquent, si notre but est de gnrer un modle pour faire de la prdiction, un lag de 1 est parfait car c'est le modle qui nous permet au mieux de grer performance et overfitting. Mais si notre but est d'expliquer, et que l'autocorrlation est un problme pour notre analyse conomique, alors il faudrait choisir un autre moyen pour slectionner le bon ordre pour un VAR.

3). Simulate impact¶

Mme si nous allons simuler l'impact de taux de croissance ngatifs avec un VAR d'ordre 1, nous avons cherch gnraliser nos fonctions tous les VAR possibles.

Le problme, c'est que plus on modlise de lag, plus la construction d'une fonction de rponse gnralise se complexifie. En effet, on pourrait imaginer des chocs diffrents sur plusieurs priodes comme par exemple au premier et au dernier trimestre. Dans notre cas, vu que le choc n'arrive qu'en t, nous n'avons pas besoin d'aller aussi loin dans la construction, mais nous avons quand mme propos une solution ce problme.

La solution que nous avons trouv est de tranformer n'importe quel VAR p en un VAR d'ordre 1. Voici comment se prsente notre transformation : Si on a $X_t$ nos n sries de donnes, $\phi_p$ notre matrice de coefficients associe aux sries laggues p priodes et $\epsilon_t$ le vecteur d'erreur de prdiction, exprims comme suit:

$$X_t= \begin{pmatrix} chine\_growth_t\\ USA\_growth_t\\ Zone\_growth_t \end{pmatrix}$$

et $$\phi_{p}=\begin{pmatrix} \phi_{1,1,p}&\phi_{1,2,p}&...&\phi_{1,n,p}\\ \phi_{2,1,p}&\phi_{2,2,p}&...&\phi_{2,n,p}\\ ...&...&...&...\\ \phi_{n,1,p}&...&...&\phi_{n,n,p} \end{pmatrix}$$ et $$\epsilon_{t}=\begin{pmatrix} \epsilon_{chine_t}\\ \epsilon_{usa_t}\\ \epsilon_{zone_t} \end{pmatrix}$$ $$$$ alors notre reprsentation d'un VAR p en VAR 1 se fait de la manire suivante: $$$$ $$\begin{pmatrix} X_t\\ X_{t-1}\\ ...\\ ...\\ X_{t-p+1} \end{pmatrix} =\begin{pmatrix} \phi_{1}&\phi_{2}&...&...&...&\phi_{p}\\ 1\\ 0&1&...\\ ...\\ 0&...&...&1&...&0 \end{pmatrix} \begin{pmatrix} X_{t-1}\\ X_{t-2}\\ ...\\ ...\\ X_{t-p} \end{pmatrix} +\begin{pmatrix} \epsilon{t}\\ 0\\ ...\\ ...\\ 0 \end{pmatrix}$$ $$$$ Etant donn que l'on sait construire la fonction de rponse gnralise partir d'une matrice $phi_1$ dans le cas d'un VAR 1, on peut gnraliser sa construction avec cette matrice (ci-dessus). Avec cette construction on pourra alors simuler des chocs sur plusieurs priodes la fois. On a donc dvelopp cette manire d'exprimer un VAR d'ordre p. Nanmoins, nous n'avons pas fait en sorte de pouvoir gnrer des chocs sur plus d'une priode. Ainsi, pour construire la fonction de rponse, nous avons uniquement utilis $\phi_1$ tant donn que le choc apparat la premire priode.

transformation_Xt_varp= function(X, p_order){
    # Permet de transformer le dataset initial pour permettre la permutation d'un VAR p  un VAR 1
        # X dsigne notre dataset
        # p_ordre dsigne l'ordre du VAR
    n= ncol(X)
    m= nrow(X)
    
    # Si on a un VAR 1, alors on ne change pas notre dataset
    if (p_order == 1){
        return (X)
    }
    else {
        tested= X
        stocked= X[-c((m-p_order+1):m), ] # srie initiale Xt
        # Le but est de pouvoir coller les sries de donnes Xt, Xt-1, ... , Xt-p
        # On a donc un dataset de dimension (m-p)x(n*p)
        for (i in c(1:p_order)){
            tested_copy= tested[-c(1:i), ]
            ajout_p_col= tested_copy[-c((m-p_order+1):m), ]
            
            stocked= cbind(ajout_p_col, stocked)
        }
        return(stocked)
    } 
}

    m= nrow(X)

    estimation_good_var= estimation_auto(X= X, p= p_order, num= n)
    para= estimation_good_var$par
    
    # On ne fait pas de transformation si on a un VAR d'ordre 1
    # On retourne uniquement les paramtres
    if (p_order == 1){
        phi_uno= matrix(para[(n+1):length(para)],n,n)
        return (phi_uno)
    }
    else {
        # Assemblage des coefficients
        # On va stack de manire horizontale les matrices de coef phi 1  phi p
        stock_phi= matrix(numeric(n*n), n, n)
        for (i in 1:p_order){
            phi_i= matrix(para[((n*n*i + n) -(n*n) + 1):(n*n*i + n)], n ,n)
            stock_phi= rbind(stock_phi, phi_i)
        }

        stock_phi= stock_phi[-c(1:n), ]
        # On va combler les trous pour fiter notre matrice de coef aux nouveaux set donnes
        # calcul transformation_Xt_varp
        # La matrice de coef sera de dimension (n*p)x(n*p)
        dim_n_p= n*p_order
        identity_mat= diag(dim_n_p-n) # permet lors du calcul X . phi d'afficher
        # Xt-1 = Xt-1 , Xt-2 = Xt-2
        zero_mat_ligne= matrix(numeric((dim_n_p - n)*n), n, (dim_n_p - n))

        stock_phi= cbind(stock_phi, rbind(identity_mat, zero_mat_ligne))

        return (stock_phi)
    }
}

phi_zero_transforma= function(X, p_order){
    # Permet de redfinir la matrice phi 0 (intercept) avec le dataset de transformation_Xt_varp
        # X correspond au dataset (dimention m*n)
        # p_order correspond  l'odre du VAR
    
    # Cas particulier dans le cas d'un VAR 1, on ne fait pas de transformation
    if (p_order == 1){
        return (phi_zero_compute(X, 1))
    }
    else {
        phi_zero= phi_zero_compute(X, p_order)
        phi_matrice= phi_transforma(X, p_order)
        diff_dim_col= ncol(phi_matrice) - length(phi_zero)
        # On comble la matrice avec des 0  gauche
        zero_comble= matrix(numeric((diff_dim_col*nrow(phi_matrice)))
                                     , nrow(phi_matrice), diff_dim_col)
        
        phi_zero_new= cbind(matrix(phi_zero, nrow(phi_matrice), length(phi_zero), byrow= T)
                            , zero_comble)
        
        return (phi_zero_new)

    }
}

block_bootstrap= function(X, ic){
    # Permet de gnrer un chantillon block bootstrapped
    # Le block bootstrapping permet de faire du bootstrapping sur une srie, et donc en rcuprant
    # des lignes proches.
    # Avec un bootstrap classique, la srie temp perdrait sa notion de temps et d'autocorr (lien intertemporel)
        # X dsigne notre dataset
        # ic dsigne la part des lignes du dataset que l'on va prendre 
        # pour gnrer ce block bootstrapped
    m= nrow(X)
    n= ncol(X)
    ic= 95
    num= floor(m*ic/100)

    # permet de block bootstrap ma srie
    stocked_data= matrix(0, 1, n, byrow= T)
    for (i in 1:num){
        # On va rcuprer 2 lignes avant et aprs la ligne que l'on a slectionn et les ajouter
        #  la suite
        random_id= floor(runif(1, 3, m - 2)) # on commence  3 car on rcupre 2 index en amont
                                             # et on finit  m - 2 car on rcupre 2 lignes 
                                             # aprs l'id slectionn alatoirement
        before_1= random_id -2
        before= random_id - 1
        after= random_id + 1
        after_1= random_id + 2

        select_val= c(before_1, before, random_id, after, after_1)
        data_sample= X[select_val, ]

        stocked_data= rbind(stocked_data, data_sample)
    }
    stocked_data= stocked_data[-1, ] # supprime la premire ligne
    
    return (stocked_data)
}
